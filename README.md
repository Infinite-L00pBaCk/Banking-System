## вңЁ Overview

> *"Not just a project вҖ” a fully working bank in your terminal."*

The **Banking Management System** simulates real-world banking operations through an interactive console interface. Every feature you'd expect from a bank вҖ” account creation, deposits, withdrawals, fund transfers, and full transaction history вҖ” all backed by **persistent file storage** that survives program restarts.

Built with pure **C++11**, zero external dependencies, and clean object-oriented architecture. Developed as an internship task for **Pinnacle Labs вҖ” May 2026**.

## рҹҡҖ Features

| # | Feature | Description |
|---|---------|-------------|
| 1 | рҹҶ• **Create Account** | Register with full name, 4-digit PIN (with confirmation), minimum вӮ№500 initial deposit. Auto-generates `PNB-XXXX` account numbers. |
| 2 | рҹ‘Ө **View Account Details** | PIN-authenticated display of complete account info. |
| 3 | рҹ’° **Deposit Funds** | Add money to any active account with full input validation. |
| 4 | рҹ’ё **Withdraw Funds** | PIN-authenticated withdrawal with insufficient balance protection. |
| 5 | рҹ”„ **Fund Transfer** | Transfer between two accounts with **dual transaction logging** for both parties. |
| 6 | рҹ“Ҡ **Check Balance** | Instant PIN-secured balance enquiry. |
| 7 | рҹ§ҫ **Transaction History** | Full timestamped audit trail вҖ” deposits, withdrawals, transfers. |
| 8 | рҹ“Ӣ **List All Accounts** | Tabular overview of every registered account. |
| 9 | вқҢ **Close Account** | Deactivate with PIN confirmation; remaining balance is returned. |
| 10 | рҹ’ҫ **File Persistence** | Data survives restarts via `.dat` flat-file storage вҖ” no database needed. |

## рҹ§  OOP Concepts

<details>
<summary><b>рҹ”’ Encapsulation</b></summary>

All data members are `private`. Balance, PIN, and active status are only accessible through validated public methods вҖ” you can never set a negative balance or bypass PIN checks from outside the class.

</details>

<details>
<summary><b>рҹҺӯ Abstraction</b></summary>

`main.cpp` calls `bank.deposit()` вҖ” it has no idea about file I/O, account lookup, or transaction logging. The entire persistence layer is invisible to the UI.

</details>

<details>
<summary><b>рҹ§© Separation of Concerns</b></summary>

| Class | Knows about |
|-------|-------------|
| `BankAccount` | One account's data and operations |
| `Transaction` | One transaction record |
| `FileManager` | All file reads and writes |
| `Bank` | Coordination and business rules |
| `main.cpp` | User interface only |

</details>

<details>
<summary><b>рҹҸ·пёҸ Type-Safe Enumerations</b></summary>

```cpp
enum class TransactionType {
    DEPOSIT, WITHDRAWAL, TRANSFER_IN, TRANSFER_OUT, ACCOUNT_CREATION
};
```

Scoped `enum class` prevents accidental integer comparisons and implicit conversions.

</details>

## рҹ’ҫ Data Storage

No SQL, no external libraries вҖ” just human-readable pipe-delimited `.dat` files.

**`accounts.dat`**
```
AccountNo | Name | PIN | Balance | Active(1/0) | CreationDate
PNB-1001|Priyam Prakash|1234|5000.00|1|2026-05-07 19:08:00
```

**`transactions.dat`**
```
AccountNo | Type | Amount | BalanceAfter | Timestamp | Description
PNB-1001|DEPOSIT|2000.00|7000.00|2026-05-07 19:10:00|Cash deposit
PNB-1001|TRANSFER_OUT|500.00|6500.00|2026-05-07 19:15:00|Transfer to PNB-1002
PNB-1002|TRANSFER_IN|500.00|1000.00|2026-05-07 19:15:00|Transfer from PNB-1001
```

> Fund transfers write **two records** вҖ” one `TRANSFER_OUT` for sender, one `TRANSFER_IN` for recipient. Complete audit trail for both accounts.

1. Select `[1]` вҶ’ Enter your full name
